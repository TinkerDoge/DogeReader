# CrossPoint Tarot Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement a standalone Tarot Card app in CrossPoint Reader that behaves like a physical deck, with custom BMP support and history tracking.

**Architecture:** A new `TarotActivity` will manage the UI and deck state. Shuffling and asset loading will be handled by modular components (`TarotDeck` and `TarotAssets`) to maintain clean separation and minimize RAM usage.

**Tech Stack:** C++, PlatformIO, ArduinoJson, SDCardManager, EInkDisplay SDK.
